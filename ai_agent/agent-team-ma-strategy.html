Jason's Notes

MA 黃金交叉策略回測系統 - Agent Team 開發計畫

專案目標

建立 Agent Team 開發 MA 黃金交叉策略回測系統

目錄結構

ma-golden-cross/
├── backend/
│   ├── data/fetcher.py       # yfinance 抓股数据
│   ├── indicators/ma.py      # MA 指標計算
│   ├── backtest/engine.py    # 回測引擎
│   ├── api/routes.py         # API 路由
│   ├── api/schemas.py        # Request/Response 格式
│   ├── tests/                # 單元測試 (pytest)
│   └── requirements.txt
├── frontend/index.html       # Chart.js K線+MA線
├── e2e/tests/               # Playwright E2E 測試
└── logs/                    # Agent 對話紀錄

團隊成員

成員角色輸出依賴
0架構師docs/architecture.md無（最先執行）
1後端工程師fetcher, ma, engine成員0
2API 工程師routes, schemas成員0+1
3前端工程師index.html成員0（並行）
4測試工程師pytest 測試成員1+2
5E2E 驗證Playwright 測試全部完成後

執行流程

Phase 0: 成員0 (架構) → 輸出 architecture.md
Phase 1: 成員1 (後端) + 成員3 (前端) [並行]
Phase 2: 成員2 (API) + 前端串接
Phase 3: 成員4 (測試)
Phase 4: 成員5 (E2E 驗證) ← 最終關卡

技術規格

  • 數據: yfinance（台股 2330.TW、美股 AAPL）
  • 後端: 純 Python MA 計算 + Flask API
  • 前端: Chart.js（K線、MA線、交叉標記）
  • 測試: pytest（單元）+ Playwright（E2E）

策略參數

參數值
短期 MA5 日
長期 MA20 日
買入訊號短期上穿長期（黃金交叉）
賣出訊號短期下穿長期（死亡交叉）

驗收標準

  1. yfinance 可抓取台/美股數據
  2. MA 計算正確
  3. 黃金/死亡交叉判斷正確
  4. 回測結果格式正確
  5. API 可正常呼叫
  6. 前端顯示 K線、MA線、交叉點
  7. 單元測試全部通過
  8. E2E 測試全部通過（Playwright）
