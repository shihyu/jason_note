## 📝 關鍵決策記錄

### 決策 1: OCR 引擎選擇
- **選項**: OCREngine (舊版) vs PriceTagOCR (改進版) vs PaddleOCR
- **選擇**: PriceTagOCR（改進版）
- **原因**:
  - Tesseract 改進版：98.4% 準確度
  - PaddleOCR：0% 準確度（不適合小區域）
  - 舊版 OCREngine：功能不足

### 決策 2: 準確度測試方式
- **選項**: pytest vs 獨立腳本
- **選擇**: 獨立腳本（`scripts/verify_accuracy.py`）
- **原因**:
  - pytest 環境無法載入舊版 ultralytics 模型
  - 獨立腳本可以繞過模組路徑問題
  - 實際驗證有效

### 決策 3: 測試檔案精簡
- **行動**: 移除 test_ocr_engine.py（舊版 API）
- **原因**:
  - 測試針對舊版 OCREngine 類別
  - 新版 PriceTagOCR 有不同的 API
  - test_accuracy.py 已全面測試 OCR 功能

### 決策 4: 資料保留策略
- **保留**:
  - videos/ (655MB) - 可重新生成影格
  - manual_labels/ (353MB) - 準確度驗證基準
  - dataset/ (22MB) - 訓練資料
  - models/ (6MB) - 訓練好的模型
- **移除**:
  - images/ (6.7GB) - 可重新生成
  - detections/ (4.9GB) - 測試產物
  - detections_v2/ (6.7GB) - 測試產物
  - failure_analysis/ (1.4MB) - 分析資料
