# 任務：將 Markdown 檔案內容轉換為台灣技術術語

## 目標
將專案中所有 Markdown (`.md`) 檔案的技術術語從中國大陸慣用語轉換為台灣慣用語（繁體中文）。

## 專案結構組織
- **專案資料夾**: `term-converter` (邏輯上的分組，實際操作在原目錄進行)
- **工具**: Python 腳本 (用於批次替換)

## 預期產出
- 所有 `.md` 檔案內容均已更新為台灣技術術語。
- 無新增永久性程式碼檔案（腳本使用後刪除）。

## Makefile 規範 (執行腳本用)
雖然主要透過 Python 腳本執行，但定義 Makefile 以符合規範：
- `make run`: 執行轉換腳本
- `make clean`: 清理臨時腳本

## 執行步驟
1. **準備腳本**: 撰寫 `convert_terms.py`，內含完整的術語對照表與遞迴檔案處理邏輯。
2. **執行轉換**: 執行 Python 腳本處理所有 `.md` 檔案。
3. **驗證**: 檢查部分檔案確認轉換是否正確。
4. **清理**: 刪除 `convert_terms.py`。
