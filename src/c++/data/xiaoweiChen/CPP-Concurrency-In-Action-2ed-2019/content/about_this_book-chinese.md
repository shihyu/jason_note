# 關於本書

本書是使用併發和多線程的指導書籍，並基於C++最新標準(11，14，17)。從最基本的`std::thread`，`std::mutex`和`std::async`的使用，到複雜的原子操作和內存模型。

## 路線圖

前4章，介紹了標準庫中的各種工具，展示使用方法。

第5章，涵蓋了內存模型和原子操作，包括原子操作如何對執行順序進行限制(這章標誌著介紹部分的結束)。

第6、7章，開始討論高級主題，如何使用基本工具去構建複雜的數據結構——第6章是基於鎖的數據結構，第7章是無鎖數據結構。

第8章，針對設計多線程代碼給了一些指導意見，覆蓋了性能問題和並行算法。

第9章，線程管理——線程池，工作隊列和中斷操作。

第10章，介紹C++17中標準庫算法對並行性的支持。

第11章，測試和調試——Bug類型，定位Bug的技巧，以及如何進行測試等等。

附錄，包括新標準中語言特性的簡要描述，主要是與多線程相關的特性，以及在第4章中提到的消息傳遞庫的實現細節和C++17線程庫的完整的參考。

## 適讀人群

如果你正在用C++寫一個多線程程序，可以閱讀本書。如果你正在使用C++標準庫中新的多線程工具，可以從本書中得到一些指導。如果你正在使用其他線程庫，本書的建議和技術指導也很值得參考。

閱讀本書需要有較好的C++基礎，關於多線程編程的知識或者經驗不是必須的。

### 如何閱讀本書

如果從來沒有寫過多線程代碼，我建議從頭到尾閱讀本書，可以跳過第5章中的較為細節的部分。

第7章內容依賴於第5章中的內容，如果跳過了第5章，應該在讀第7章前，讀一下第5章。

如果沒有用過C++11，可以先看一下附錄，當在正文中遇到一些沒見過的工具或特性時，可以隨時回看附錄。

即使有不同環境下寫多線程代碼的經驗，開始的章節仍有必要瀏覽一下，這樣就能清楚地知道，你所熟知的工具在C++的新標準中對應於哪些工具。如果使用原子變量去做一些底層工作，第5章必讀。第8章有關C++多線程的異常和安全性的內容很值得一看。如果對某些關鍵詞比較感興趣，索引和目錄能夠幫你快速找到相應的內容。

你可能喜歡回顧主要的章節，並用自己的方式閱讀示例代碼。雖然你已經瞭解C++線程庫，但附錄D還是很有用。例如，查閱每個類和函數的細節。

## 代碼公約和下載

為了區分普通文本，清單和正文中的所有代碼都採用`像這樣的固定寬度的字體`。許多清單都伴隨著代碼註釋，突出顯示重要的概念。在某些情況下，你可以通過頁下給出的快捷鏈接進行查閱。

本書所有實例的源代碼，可在出版商的網站上進行下載：www.manning.com/books/c-plus-plus-concurrency-in-action-second-edition。

你也可以從github上下載源碼：https://github.com/anthonywilliams/ccia_code_samples。

### 軟件需求

使用書中的代碼，需要一個較新的C++編譯器(要支持C++17語言的特性(見附錄A))，還需要C++支持標準線程庫。

寫本書的時候，最新版本的g++、clang++和Microsoft Visual Studio都對C++17的標準現成庫進行了實現。他們也會支持附錄中的大多數語言特性，那些目前還不被支持的特性也會逐漸被支持。

我的公司Software Solutions Ltd，銷售C++11標準線程庫的完整實現，其可以使用在一些舊編譯器上，以及為新版本的clang、gcc和Microsoft Visual Studio實現的併發技術標準[1]。這個線程庫也可以用來測試本書中的例子。

Boost線程庫[2]提供的API，已經可移植到多個平臺。本書中的大多數例子將`std::`替換為`boost::`，再`#include`引用相應的頭文件，就能使用Boost線程庫來運行。新標準中，部分編譯器可能還不支持(例如`std::async`)，或在Boost線程庫中有著不同名字(例如：`boost::unique_future`)。
