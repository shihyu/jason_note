\begin{enumerate}
\item
什么是质量属性？

\begin{itemize}
\item 
系统可能具有的特征或品质通常称为“能力”，因为它们中的许多名字中都有这个后缀，例如：可移植性。
\end{itemize}

\item
收集需求时应该使用哪些资源？

\begin{itemize}
\item 
系统的上下文、现有的文档和系统的相关方。
\end{itemize}

\item
如何判断某个需求在架构上是否重要？

\begin{itemize}
\item 
架构上的重要需求(ASR)通常需要一个单独的软件组件，影响系统的大部分，很难实现，并且/或迫使开发者做出权衡。
\end{itemize}

\item
应该如何以图形化的方式记录各方对系统的功能性需求？

\begin{itemize}
\item 
准备一个用例图。
\end{itemize}

\item
何时开发视图文档？

\begin{itemize}
\item 
如果正在开发一个带有许多模块的大型系统，并且需要与所有软件团队沟通全局约束和通用设计选择。
\end{itemize}

\item
如何自动检查代码的API文档是否过期？

\begin{itemize}
\item 
Doxygen有内置的检查，比如在注释中警告函数签名和参数不匹配
\end{itemize}

\item
如何在图中显示，特定的操作由系统的不同组件处理？

\begin{itemize}
\item 
为此目的使用UML交互图之一。序列图是一个很好的选择，通信图在某些场景中也可以很好工作。
\end{itemize}

\end{enumerate}
