# 第二章：信息的表示和處理

整數：
- 無符號整數直接用原碼錶示。
- 有符號整數通常用補碼錶示。
- 計算機中的整數運算可能會溢出，溢出後存不下的高位被捨棄，表現為結果對`2^32/64`取模。

浮點數：
- 浮點數使用二進制科學計數法近似表示一個實數，現代的機器中通常都支持浮點數的IEEE754標準。
- 浮點數由一個符號位、多個階碼（二進制指數，`float`8位，`double`11位）和多個尾數位（二進制精度位，`float`23位，`double`52位，原碼錶示）構成。
- 浮點數的二進制表示有四種情況：規格化的、非規格化的、無窮大（分正負）、NaN。
    - 通常來說一個正常值是規格化的。
    - 非規格化的值提供了表示`0.0`的方法（精度位可以為`0 1`，所以存在`+0.0 -0.0`，一般來說沒有區別，但某些情況可能有影響，比如除零時無窮大的符號）。
    - 無窮大出現在上溢的情況中，比如浮點乘法上溢出、除零等情況，也分正負。
    - NaN是Not A Number的意思，出現在比如無窮大與無窮大的運算結果，NaN與其他數的運算結果。
    - 由於NaN的存在，浮點數的比較是偏序而不是全序的。
- 在最普遍的`int`32位，`float`32位，`double`64位的環境中：
    - `int`轉換為`float`不會溢出，但可能會有舍入。
    - `int float`轉換為`double`不會有溢出或者舍入。
    - `double`轉`float`，可能溢出為無窮大，可能有舍入。
    - `float double`轉`int`，值會向零舍入，可能會溢出。
- 因為有舍入並且可能溢出，浮點數的計算不具有結合性，這在某些時候可能會影響程序正確性，需要意識到。
