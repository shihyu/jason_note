# 章節

此檔案定義所有章節、其排序、影響等級和描述。
章節 ID（括號中）是用於分組規則的檔名前綴。

---

## 1. 消除瀑布流 (async)

**影響：** 關鍵
**描述：** 瀑布流是效能殺手第一名。每個循序的 await 都會增加完整的網路延遲。消除它們能帶來最大的效能提升。

## 2. 打包大小優化 (bundle)

**影響：** 關鍵
**描述：** 減少初始打包大小可改善可互動時間（Time to Interactive）和最大內容繪製（Largest Contentful Paint）。

## 3. 伺服器端效能 (server)

**影響：** 高
**描述：** 優化伺服器端渲染和資料擷取可消除伺服器端瀑布流並減少回應時間。

## 4. 客戶端資料擷取 (client)

**影響：** 中高
**描述：** 自動去重和高效的資料擷取模式可減少冗餘的網路請求。

## 5. 重新渲染優化 (rerender)

**影響：** 中
**描述：** 減少不必要的重新渲染可最小化浪費的計算並改善 UI 回應性。

## 6. 渲染效能 (rendering)

**影響：** 中
**描述：** 優化渲染過程可減少瀏覽器需要做的工作。

## 7. JavaScript 效能 (js)

**影響：** 低中
**描述：** 熱點路徑的微優化可以累積成有意義的改善。

## 8. 進階模式 (advanced)

**影響：** 低
**描述：** 需要謹慎實作的特定案例進階模式。
