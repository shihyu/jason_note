## 快速開始

### 第一步：打開知識庫

```
位置：/home/shihyu/github/jason_note/src/ai_agent/llm-wiki
```

在 Obsidian 中開啟這個資料夾作為 vault。

### 第二步：放入來源文章

將你有興趣的教學文章、筆記轉為 Markdown，放入 `sources/` 資料夾。

### 第三步：告訴 LLM 處理

```
請處理 sources/你的文章.md
```

LLM 會自動更新整個知識庫。

## 工作流程

### 攝取（Ingest）— 新知識進來

```
你：把文章放入 sources/
       ↓
你：告訴 LLM「請處理 sources/xxx.md」
       ↓
LLM：
  1. 讀取來源文章
  2. 識別關鍵概念
  3. 更新 wiki/ 中的相關頁面
  4. 建立新的概念頁面（如有必要）
  5. 更新 wiki/index.md
  6. 在 wiki/log.md 記錄操作
```

### 查詢（Query）— 問問題

```
你：「比較 Rust 和 Go 的錯誤處理方式」
       ↓
LLM：
  1. 搜尋 wiki/ 中的相關頁面
  2. 閱讀並綜合整理
  3. 回答你的問題
       ↓
你：（可選）「把這個回答存到 wiki」
       ↓
LLM：建立新頁面存入知識庫
```

### 整理（Lint）— 健康檢查

```
你：「幫我整理 wiki」
       ↓
LLM：
  1. 檢查過時的頁面
  2. 標記孤立的頁面
  3. 找出缺失的交叉引用
  4. 報告問題
```

## Obsidian + LLM 搭配使用

### 推薦配置

```
+--------------------------------------------------+
|                                                  |
|   Obsidian（左側）          OpenCode（右側）      |
|                                                  |
|   你即時瀏覽 Wiki          LLM 處理知識庫         |
|                                                  |
|   - 圖譜檢視               - 讀取來源            |
|   - 追蹤連結               - 更新頁面            |
|   - 閱讀更新               - 維護 index          |
|                                                  |
+--------------------------------------------------+
```

### Obsidian 推薦外掛

| 外掛 | 用途 |
|------|------|
| Web Clipper | 網頁轉 Markdown，快速抓來源 |
| Graph View | 視覺化頁面之間的連結 |
| Dataview | 查詢頁面 metadata |
| Templater | 自動化建立新頁面格式 |

### 下載圖片到本地

1. 設定 → 檔案與連結 → 附件資料夾設為 `assets/`
2. 設定 → 快捷鍵 → 搜尋「Download」
3. 綁定快捷鍵（如 `Ctrl+Shift+D`）
4. 讀文章時按快捷鍵，圖片會下載到本地

## sources 資料夾

### 放什麼

- 網路上覺得有價值的教學文章
- 自己寫的學習筆記
- 論文、文件、書籍摘錄
- 任何你想記住的程式知識

### 檔案命名建議

```
sources/
├── 2026-04-07-rust-所有權.md      # 日期-語言-主題
├── 2026-04-06-go-併發入門.md
├── 2026-04-05-python-async.md
└── 2026-04-04-cpp-智慧指標.md
```

### 如何獲取 Markdown

**方法一：Obsidian Web Clipper**
- 安裝瀏覽器擴充
- 瀏覽任何網頁，點擊圖示轉為 Markdown

**方法二：複製貼上**
- 複製網頁文字
- 貼到 Markdown 編輯器
- 手動整理格式

**方法三：LLM 幫忙**
- 丢給 LLM 一個 URL
- 說「請幫我轉成 Markdown」

## Wiki 頁面說明

### wiki/index.md — 總入口

所有知識庫內容的目錄，按語言、概念、專案分類。

### wiki/log.md — 操作日誌

所有操作的時間軸記錄。格式：

```markdown
## [2026-04-07] ingest | Rust 所有權
- 來源：sources/2026-04-07-rust-所有權.md
- 更新頁面：languages/rust, concepts/記憶體管理
```

### wiki/languages/ — 語言頁面

每個語言的整合頁面，包含：
- 語言定位與特色
- 語法速查
- 生態常用工具
- 與其他語言的比較

### wiki/concepts/ — 概念頁面

跨語言通用概念，包含：
- 定義
- 各語言的實現方式
- 比較表格

### wiki/projects/ — 專案頁面

你的專案筆記（目前為止是空的，你會需要時再建立）。

### wiki/quick-ref/ — 速查表

常用語法、指令的快速參考。

## CLAUDE.md 是什麼

`CLAUDE.md` 是給 LLM 的**工作手冊**。

當 LLM 在這個資料夾工作時，它會參考這份文件來了解：
- Wiki 的結構
- 頁面格式規範
- 工作流程
- 什麼該做，什麼不該做

### 你可以修改它

隨著你使用一段時間，會發現某些規則需要調整。那時你可以修改 `CLAUDE.md`，讓 LLM 符合你的工作方式。

## 常見問題

### Q：需要網路嗎？

A：知識庫是純本地 Markdown 檔案，離線也能用。LLM 需要網路。

### Q：這個和 RAG 有什麼不同？

A：
| RAG | LLM-Wiki |
|-----|----------|
| 每次查詢重新檢索 | 知識已建立並連結 |
| 沒有累積 | 持續複利增長 |
| 答案片段散落 | 答案經過綜合整理 |

### Q：Obsidian 和普通資料夾哪個好？

A：Obsidian 有圖譜檢視、搜尋、外掛，對中型知識庫很有幫助。如果只是幾十個頁面，普通編輯器也夠用。

### Q：可以一次處理多個來源嗎？

A：可以，但建議一次一個。保持參與能讓你引導 LLM 強調重要的部分。

### Q：Wiki 會不會越來越大很難管理？

A：靠 `wiki/index.md` 導航，加上 `wiki/log.md` 了解最近變化。可以用 `grep "^## \[" wiki/log.md | tail -10` 快速看最近記錄。

### Q：可以把現有的筆記迁入吗？

A：可以。把現有 Markdown 丢進 `sources/`，告訴 LLM 處理，LLM 會幫你整合到 Wiki 中。
