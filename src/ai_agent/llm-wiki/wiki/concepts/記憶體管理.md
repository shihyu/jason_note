# 記憶體管理

## 定義

> 記憶體管理是指程式如何配置、使用、釋放記憶體的機制。

## 各語言的記憶體管理策略

### Rust：所有權系統（無 GC）

**核心概念**：
- 每個值有一個擁有者（owner）
- 一次只有一個擁有者
- 擁有者離開作用域時，值被 drop

```rust
fn main() {
    let s1 = String::from("hello");  // s1 擁有這個 String
    let s2 = s1;                      // 擁有權轉移給 s2
    // println!("{}", s1);  // 錯誤！s1 已不再有效
    
    let s3 = s2.clone();             // 明確複製（深拷貝）
    
    let x = 5;
    let y = x;                        // Copy trait，i32 是複製而非移動
    println!("{} {}", x, y);          // 沒問題
}
```

**借用（Borrowing）**：
```rust
fn calculate(s: &String) -> usize {
    s.len()
}  // s 離開作用域，但不擁有值，所以不會 drop

let s = String::from("hello");
let len = calculate(&s);  // 借用
println!("{} has {} chars", s, len);  // s 仍然有效
```

### Go：GC（全自動）

**特點**：
- 記憶體配置：`new()` 或 `make()`
- 自動 GC 回收不再使用的記憶體
- 逃逸分析（escape analysis）優化堆疊配置

```go
func main() {
    // 堆配置（GC 回收）
    s := make([]int, 0, 10)
    s = append(s, 1, 2, 3)
    
    // 指標
    p := new(int)  // *int
    *p = 42
    
    // 參考語意 vs 值語意
    type Point struct {
        X, Y int
    }
    
    p1 := Point{1, 2}
    p2 := p1    // 複製
    p2.X = 3    // p1 不受影響
    
    p3 := &Point{1, 2}  // 指標
    p4 := p3            // 複製指標
    p4.X = 3            // p3 也受影響
}
```

### Python：GC（自動引用計數 + 標記清除）

**特點**：
- 大部分時間 GC 自動處理
- 可使用 `__del__` 自訂資源清理（但要小心）
- 迴圈引用需要 GC 處理

```python
import gc

class MyClass:
    def __init__(self, name):
        self.name = name
        print(f"Created {self.name}")
    
    def __del__(self):
        print(f"Deleted {self.name}")

# 引用計數為 0 時立即刪除
obj = MyClass("test")
del obj  # __del__ 被調用

# 手動 GC
gc.collect()
```

### JavaScript：GC（全自動）

**特點**：
- 幾乎不用關心記憶體釋放
- 避免全域變數累積
- 小心閉包造成的記憶體洩漏

```javascript
// 閉包保留記憶體
function outer() {
    const bigArray = new Array(1000000);
    
    return function inner() {
        return bigArray.length;  // bigArray 不會被 GC 回收
    };
}

// 手動釋放
let obj = { /* ... */ };
obj = null;  // 幫助 GC
```

### C++：手動 + RAII + 智慧指標

**方式**：
1. 手動 `new`/`delete`（不推薦）
2. RAII（推薦）
3. 智慧指標（現代 C++）

```cpp
// 手動（危險！）
int* p = new int(42);
// ... 使用 p
delete p;  // 容易忘記或錯誤

// RAII - 類別管理資源
class File {
    FILE* f;
public:
    File(const char* name) {
        f = fopen(name, "r");
    }
    ~File() {
        if (f) fclose(f);  // 解構時自動關閉
    }
};

// 智慧指標（C++11）
#include <memory>

auto uptr = std::make_unique<int>(42);     // 獨佔
auto sptr = std::make_shared<int>(42);     // 共享

// weak_ptr 打破循環引用
auto sptr1 = std::make_shared<Parent>();
auto sptr2 = std::make_shared<Child>();
sptr1->child = sptr2;
sptr2->parent = sptr1;  // 循環引用！

// 使用 weak_ptr 解決
class Child {
public:
    std::weak_ptr<Parent> parent;
};
```

## 比較摘要

| 語言 | 策略 | 手動干預 | 優點 | 缺點 |
|------|------|----------|------|------|
| Rust | 所有權 + borrowing | 編譯時檢查 | 無 GC 暫停、安全 | 學習曲線陡 |
| Go | GC | 幾乎無 | 簡單易用 | GC 暫停 |
| Python | 引用計數 + GC | 可選 | 開發者友好 | 效能開銷 |
| JavaScript | GC | 很少 | 簡單 | GC 可能卡顿 |
| C++ | 手動 + RAII + 智慧指標 | 可選 | 效能完全可控 | 複雜度 |

## 常見記憶體錯誤

| 錯誤類型 | 語言 | 說明 |
|----------|------|------|
| 記憶體洩漏 | C++, JS, Python | 忘記釋放或持有不需要的引用 |
| 垂指標 | C++ | 使用已釋放的記憶體 |
| 緩衝區溢位 | C++ | 寫入超出邊界 |
| 資料競爭 | 多執行緒 | 多執行緒同時修改共享記憶體 |
| 雙重釋放 | C++ | 釋放已經釋放的記憶體 |

## 實踐建議

### Rust
- 善用借用檢查器
- `Clone` 當你不確定時
- `Rc<RefCell<T>>` 當你需要共享可變狀態

### Go
- 用 `p := &Point{}` 代替 `p := new(Point)`
- 留意逃逸分析結果
- 大型資料用指標傳遞

### Python
- 避免在全域存放大型物件
- 及時刪除或設 `None`
- 迴圈引用時使用 `gc.collect()`

### C++
- 優先用智慧指標而非 `new`/`delete`
- RAII 設計類別
- 用 `std::vector` 代替原始陣列
