# 設計模式

## 定義

> 設計模式是針對軟體設計中常見問題的可重用解決方案。

## 三大類型

1. **Creational**：建立物件的方式
2. **Structural**：組合類別/物件的方式
3. **Behavioral**：物件之間的責任分配

---

## 創建型模式（Creational）

### 單例模式（Singleton）

**目的**：確保一個類別只有一個實例。

```rust
// Rust: 懶惰單例
use std::sync::Mutex;

struct Database {
    connection: String,
}

impl Database {
    fn new() -> Self {
        Database { connection: "localhost".to_string() }
    }
}

lazy_static::lazy_static! {
    static ref DB: Mutex<Database> = Mutex::new(Database::new());
}

// 使用
let db = DB.lock().unwrap();
```

```go
// Go: sync.Once
type DB struct {
    connection string
}

var (
    db     *DB
    once   sync.Once
)

func GetDB() *DB {
    once.Do(func() {
        db = &DB{connection: "localhost"}
    })
    return db
}
```

```python
# Python: 模組就是單例
# database.py
class Database:
    pass

db = Database()  # 模組層級只有一個

# 使用
# from database import db
```

### 工廠模式（Factory）

**目的**：將物件建立邏輯封裝。

```rust
// Rust: 泛型工廠
trait Animal {
    fn speak(&self);
}

struct Dog;
struct Cat;

impl Animal for Dog {
    fn speak(&self) { println!("Woof!"); }
}

impl Animal for Cat {
    fn speak(&self) { println!("Meow!"); }
}

enum AnimalType {
    Dog,
    Cat,
}

fn create_animal(t: AnimalType) -> Box<dyn Animal> {
    match t {
        AnimalType::Dog => Box::new(Dog),
        AnimalType::Cat => Box::new(Cat),
    }
}
```

## 結構型模式（Structural）

### 轉接器模式（Adapter）

**目的**：讓不相容的介面能夠合作。

```go
// 轉接器：讓舊介面適配新介面
type LegacyAPI struct{}

func (l *LegacyAPI) OldCalc(x, y int) int {
    return x + y
}

type NewAPI interface {
    Calculate(x, y int) int
}

type Adapter struct {
    legacy *LegacyAPI
}

func (a *Adapter) Calculate(x, y int) int {
    return a.legacy.OldCalc(x, y)
}
```

### 裝飾器模式（Decorator）

**目的**：動態新增行為。

```python
# Python: 裝飾器
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper

@timer
def slow_function():
    import time
    time.sleep(1)
```

```rust
// Rust: 裝飾器（模式匹配）
struct Circle {
    radius: f64,
}

trait Draw {
    fn draw(&self);
}

struct WithBorder<T> {
    inner: T,
    border_width: f64,
}

impl<T: Draw> Draw for WithBorder<T> {
    fn draw(&self) {
        self.inner.draw();
        println!("Border: {}px", self.border_width);
    }
}
```

## 行為型模式（Behavioral）

### 觀察者模式（Observer）

**目的**：一對多依賴關係，當一個物件改變，所有依賴者都通知。

```rust
// Rust: 事件系統
use std::rc::Rc;
use std::cell::RefCell;

trait Observer {
    fn notify(&self, event: &str);
}

struct Subject {
    observers: Vec<Box<dyn Observer>>,
}

impl Subject {
    fn new() -> Self {
        Subject { observers: Vec::new() }
    }
    
    fn add_observer(&mut self, obs: Box<dyn Observer>) {
        self.observers.push(obs);
    }
    
    fn notify_all(&self, event: &str) {
        for obs in &self.observers {
            obs.notify(event);
        }
    }
}
```

```python
# Python: Observer
class Subject:
    def __init__(self):
        self._observers = []
    
    def attach(self, observer):
        self._observers.append(observer)
    
    def notify(self, event):
        for obs in self._observers:
            obs.update(event)

class Observer:
    def update(self, event):
        print(f"Received: {event}")
```

### 策略模式（Strategy）

**目的**：能在執行時選擇演算法。

```go
// Go: 策略模式
type SortStrategy interface {
    Sort([]int) []int
}

type QuickSort struct{}
func (qs *QuickSort) Sort(data []int) []int {
    // 快速排序實現
    return data
}

type MergeSort struct{}
func (ms *MergeSort) Sort(data []int) []int {
    // 合併排序實現
    return data
}

type Sorter struct {
    strategy SortStrategy
}

func (s *Sorter) SetStrategy(strategy SortStrategy) {
    s.strategy = strategy
}

func (s *Sorter) Sort(data []int) []int {
    return s.strategy.Sort(data)
}
```

### 指令模式（Command）

**目的**：將請求封裝為物件。

```rust
// Rust: Command 模式
trait Command {
    fn execute(&self);
}

struct SaveCommand;
impl Command for SaveCommand {
    fn execute(&self) {
        println!("Saving...");
    }
}

struct Button {
    command: Box<dyn Command>,
}

impl Button {
    fn press(&self) {
        self.command.execute();
    }
}
```

## 跨語言比較

| 模式 | Rust | Go | Python |
|------|------|----|----|
| Singleton | `lazy_static` | `sync.Once` | 模組 |
| Factory | `match` + trait | 介面函式 | `__new__` |
| Observer | `trait` + `Vec` | channel + goroutine | callback list |
| Strategy | trait object | interface | function |
| Command | trait | interface | callable |

---

## 模式使用時機

| 模式 | 何時用 |
|------|--------|
| Singleton | 資料庫連線、日誌、全域設定 |
| Factory | 複雜物件建立、依類型建立不同物件 |
| Observer | 事件系統、發布-訂閱 |
| Strategy | 多種演算法可選、需動態切換 |
| Command | 佇列、復原/重做、延遲執行 |
| Adapter | 整合舊系統、相容不同介面 |
| Decorator | 動態新增功能、職責鍊 |

---

## 過度使用的警告

**不要強迫用模式！**

- 簡單程式碼 > 復雜的模式實現
- 模式是工具，不是目標
- 如果用普通函式就能解決，就不要用模式
