# 併發模型

## 定義

> 併發是指多個任務在重疊的時間段內交替執行，而非真正同時進行。

## 各語言的併發實現

### Rust

**模型**：`Send + Sync` trait + 多種並發原語

| 原語 | 用途 |
|------|------|
| `std::thread` | 原生執行緒 |
| `Arc<Mutex<T>>` | 共享記憶體 |
| `mpsc::channel` | 訊息傳遞 |
| `tokio` | async/await 生態 |

```rust
use std::sync::{Arc, Mutex};
use std::thread;

let counter = Arc::new(Mutex::new(0));
let mut handles = vec![];

for _ in 0..10 {
    let counter = Arc::clone(&counter);
    handles.push(thread::spawn(move || {
        let mut num = counter.lock().unwrap();
        *num += 1;
    }));
}

for handle in handles {
    handle.join().unwrap();
}
```

### Go

**模型**：Goroutine + Channel（CSP）

```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        fmt.Println("worker", id, "processing job", j)
        results <- j * 2
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    // 啟動 3 個 worker
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }

    // 發送任務
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs)

    // 收集結果
    for a := 1; a <= 5; a++ {
        <-results
    }
}
```

### Python

**模型**：threading / asyncio / multiprocessing

```python
import asyncio

async def fetch(url):
    await asyncio.sleep(1)  # 模擬 I/O
    return f"Result from {url}"

async def main():
    tasks = [fetch(url) for url in urls]
    results = await asyncio.gather(*tasks)

# 或使用執行緒
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=4) as executor:
    futures = [executor.submit(do_work, x) for x in items]
    results = [f.result() for f in futures]
```

### JavaScript

**模型**：Event Loop + Promise/async-await

```javascript
// Node.js 環境
const fs = require('fs').promises;

// 異步讀取
async function readFiles() {
    const data = await Promise.all([
        fs.readFile('file1.txt'),
        fs.readFile('file2.txt'),
    ]);
    return data;
}

// Web Worker（真正的多執行緒）
const worker = new Worker('worker.js');
worker.postMessage({ type: 'compute', data: bigArray });
worker.onmessage = (e) => console.log('Result:', e.data);
```

### C++

**模型**：std::thread + std::async + C++20 coroutines

```cpp
#include <thread>
#include <future>
#include <vector>

void worker(int id) {
    std::cout << "Worker " << id << " running\n";
}

int main() {
    std::vector<std::thread> threads;
    
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(worker, i);
    }
    
    for (auto& t : threads) {
        t.join();
    }
    
    // 或使用 async
    auto future = std::async(std::launch::async, []() {
        return compute();
    });
}
```

## 比較摘要

| 語言 | 併發單位 | 記憶體共享 | 通訊方式 | 難度 |
|------|----------|------------|----------|------|
| Rust | 執行緒 | Arc + Mutex | Channel | 中高 |
| Go | Goroutine | 否（靠 channel） | Channel | 低 |
| Python | 執行緒/協程 | 是（受 GIL 限制） | Queue / asyncio | 中 |
| JavaScript | 事件循環 | 是（單執行緒） | Callback / Promise | 低 |
| C++ | 執行緒 | 是（手動） | mutex / future | 高 |

## 適用場景

| 場景 | 推薦語言 |
|------|----------|
| 高效能網路服務 | Go / Rust |
| CPU 密集計算 | C++ / Rust |
| I/O 密集爬蟲 | Python (asyncio) / Go |
| 即時 Web 前端 | JavaScript |
| 嵌入式即時系統 | C / C++ / Rust |
