# 錯誤處理

## 定義

> 錯誤處理是指程式遭遇非預期情況時（網路失敗、檔案不存在、非法輸入）如何應對的機制。

## 各語言的錯誤處理方式

### Rust：Result + Option

**哲學**：錯誤是值，必須被處理。

```rust
// Result<T, E> - 可能失敗的操作
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err("cannot divide by zero".to_string())
    } else {
        Ok(a / b)
    }
}

// 使用 match
match divide(10.0, 2.0) {
    Ok(result) => println!("Result: {}", result),
    Err(e) => println!("Error: {}", e),
}

// 使用 ? 運算子（早期返回）
fn calculate() -> Result<f64, String> {
    let result = divide(10.0, 0.0)?;  // Err 時直接返回
    Ok(result * 2.0)
}

// Option<T> - 可能不存在
fn find(haystack: &str, needle: char) -> Option<usize> {
    haystack.find(needle)
}

let idx = find("hello", 'x');
if let Some(pos) = idx {
    println!("Found at {}", pos);
}

// 組合器
let result = divide(10.0, 2.0)
    .map(|v| v * 2.0)           // 轉換 Ok 值
    .or_else(|_| Ok(0.0));       // 失敗時提供預設值
```

### Go：Error

**哲學**：錯誤只是值，顯式處理。

```go
func divide(a, b float64) (float64, error) {
    if b == 0.0 {
        return 0, errors.New("cannot divide by zero")
    }
    return a / b, nil
}

func calculate() (float64, error) {
    result, err := divide(10.0, 0.0)
    if err != nil {
        return 0, err  // 顯式處理
    }
    return result * 2, nil
}

// 使用 errors.Is / As 檢查錯誤類型
if errors.Is(err, os.ErrNotExist) {
    // 檔案不存在
}

// 錯誤包裝（Go 1.13+）
fmt.Errorf("operation failed: %w", err)
```

### Python：Exception

**哲學**：發生了異常情況，向上傳播。

```python
def divide(a, b):
    if b == 0:
        raise ValueError("cannot divide by zero")
    return a / b

# 處理
try:
    result = divide(10, 0)
except ValueError as e:
    print(f"Error: {e}")
except Exception as e:
    print(f"Unexpected: {e}")
finally:
    print("Cleanup")  # 一定執行

# 自訂異常
class MyError(Exception):
    pass

# 錯誤傳播
def outer():
    try:
        inner()
    except ValueError:
        print("Caught!")
        raise  # 重新拋出
```

### JavaScript：try-catch + Promise rejection

**哲學**：同步用 try-catch，非同步用 Promise。

```javascript
// 同步
try {
    const result = JSON.parse(invalidJson);
} catch (e) {
    console.error('Parse error:', e.message);
}

// Async/Await
async function fetchData() {
    try {
        const response = await fetch(url);
        if (!response.ok) {
            throw new Error(`HTTP ${response.status}`);
        }
        return await response.json();
    } catch (e) {
        console.error('Fetch failed:', e);
        throw e;  // 重新拋出
    }
}

// Promise 鏈
fetch(url)
    .then(data => process(data))
    .catch(e => console.error(e));

// 自訂錯誤
class ApiError extends Error {
    constructor(message, status) {
        super(message);
        this.name = 'ApiError';
        this.status = status;
    }
}
```

### C++：Exception + std::optional

**哲學**：錯誤時拋異常，設計良好時才用。

```cpp
#include <stdexcept>
#include <optional>

double divide(double a, double b) {
    if (b == 0.0) {
        throw std::invalid_argument("cannot divide by zero");
    }
    return a / b;
}

try {
    double result = divide(10.0, 0.0);
} catch (const std::invalid_argument& e) {
    std::cerr << "Error: " << e.what() << std::endl;
} catch (...) {
    std::cerr << "Unknown error" << std::endl;
}

// C++17 std::optional（不拋異常的錯誤）
std::optional<double> safe_divide(double a, double b) {
    if (b == 0.0) return std::nullopt;
    return a / b;
}

auto result = safe_divide(10.0, 0.0);
if (result) {
    std::cout << *result << std::endl;
} else {
    std::cout << "Division failed" << std::endl;
}

// C++23 std::expected（更明確）
#include <expected>

std::expected<double, std::string> divide(double a, double b) {
    if (b == 0.0) return std::unexpected("cannot divide by zero");
    return a / b;
}
```

## 比較摘要

| 語言 | 機制 | 處理方式 | 適合場景 |
|------|------|----------|----------|
| Rust | Result/Option | `?` / match | 一切場景，偏好明確 |
| Go | Error | if err != nil | 簡單直接，可恢復錯誤 |
| Python | Exception | try-except | 一切場景，異常控制流 |
| JavaScript | try-catch/Promise | try-catch/async | 同步+非同步 |
| C++ | Exception/optional/expected | try-catch | 例外情況，效能關鍵用 optional |

## 錯誤處理原則

### 1. 區分可恢復 vs 不可恢復錯誤

```rust
// 可恢復：用 Result
fn read_config() -> Result<Config, io::Error> { ... }

// 不可恢復：用 panic
fn get(&self, index: usize) -> &T {
    if index >= self.len() {
        panic!("index out of bounds");  // 程式錯誤，不是使用者錯誤
    }
}
```

### 2. 錯誤往上傳播

```go
// 不要隱藏錯誤
func bad() error {
    doSomething()  // 錯誤消失了！
    return nil
}

func good() error {
    if err := doSomething(); err != nil {
        return fmt.Errorf("doSomething: %w", err)  // 包裝錯誤
    }
    return nil
}
```

### 3. 不要用異常做控制流

```python
# 不好
try:
    for item in huge_list:
        if item.id == target_id:
            raise FoundException(item)
except FoundException as e:
    return e.item

# 好
for item in huge_list:
    if item.id == target_id:
        return item
return None
```

### 4. 提供有價值的錯誤訊息

```rust
// 不好的錯誤
Err("Error")

// 好的錯誤
Err(format!("Failed to open file '{}': {}", path, io::error()))
```

## 實踐建議

| 語言 | 建議 |
|------|------|
| Rust | 多用 `?`，建立自訂錯誤類型用 `thiserror` |
| Go | 用 `errors.Is/As`，區分暫時性/永久性錯誤 |
| Python | 內建異常足夠，自訂異常時繼承 `Exception` 而非 `BaseException` |
| JavaScript | 統一用 `Error` 物件，區分網路錯誤和業務錯誤 |
| C++ | 高效能路徑用 `std::optional`，真正例外情況用 `throw` |
