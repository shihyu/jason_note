# Rust 所有權系統詳解

## 核心概念

Rust 的所有權系統是其記憶體安全的核心：

1. **每個值有一個擁有者（owner）**
2. **一次只能有一個擁有者**
3. **擁有者離開作用域時，值被釋放**

## 移動語意

```rust
let s1 = String::from("hello");
let s2 = s1;  // s1 移動到 s2
// println!("{}", s1);  // 編譯錯誤！s1 已無效
```

## 複製語意

```rust
let x = 5;
let y = x;  // i32 是 Copy 的，所以 x 仍然有效
println!("{} {}", x, y);  // 沒問題
```

## 借用

借用讓你可以使用值但不取得擁有權：

```rust
fn calculate(s: &String) -> usize {
    s.len()
}

let s = String::from("hello");
let len = calculate(&s);  // & 表示借用
println!("{} has {} chars", s, len);  // s 仍然有效
```

## 可變借用

```rust
let mut s = String::from("hello");
s.push_str(", world");  // 需要 mut 才能修改
```

## 借用規則

1. 一個值可以有**任意數量**的不可變借用（`&T`）
2. 或只有**一個**可變借用（`&mut T`）
3. 不能同時有不可變和可變借用

## Rust 與其他語言的差異

| 語言 | 策略 | 效能 | 安全性 |
|------|------|------|--------|
| Rust | 所有權系統 | 無成本 | 編譯時保證 |
| C++ | 手動 + smart ptr | 完全可控 | 依賴程式設計師 |
| Go | GC | GC 暫停 | 執行時保證 |
| Python | GC + 引用計數 | GC 開銷 | 執行時保證 |
