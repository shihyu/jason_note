# 程式開發知識庫

一個由 LLM 維護的主動式個人知識庫。

---

## 快速開始

### 1. 放入來源

將你有興趣的文章、教程、筆記轉為 Markdown，放入 `sources/` 目錄。

### 2. 告訴 LLM 處理

```
請處理 sources/你的文章.md
```

### 3. 查看結果

LLM 會幫你：
- 更新 `wiki/` 裡的相關頁面
- 建立新的概念頁面
- 維護 `wiki/index.md` 和 `wiki/log.md`

---

## 目錄結構

```

├── CLAUDE.md          # LLM 工作手冊（這就是你給 LLM 的指示）
├── README.md          # 你在這裡
│
├── sources/          # 原始來源（你放的）
│   └── ...
│
├── wiki/             # LLM 維護的知識庫
│   ├── index.md      # 總索引
│   ├── log.md        # 操作日誌
│   │
│   ├── languages/    # 語言頁面
│   │   ├── rust.md
│   │   ├── go.md
│   │   ├── python.md
│   │   ├── javascript.md
│   │   └── cpp.md
│   │
│   ├── concepts/     # 概念頁面
│   │   ├── 併發模型.md
│   │   ├── 記憶體管理.md
│   │   ├── 錯誤處理.md

```

---

## 工作流程

### 攝取（Ingest）

將新來源轉為 Markdown，放入 `sources/`，然後說：

```
請處理 sources/xxx.md
```

LLM 會：
1. 讀取來源
2. 識別關鍵概念
3. 更新 wiki/ 中所有相關頁面
4. 更新 index.md
5. 在 log.md 記錄

### 查詢（Query）

直接問問題：

```
比較 Rust 和 Go 的錯誤處理
```

LLM 會搜尋 wiki 頁面，綜合整理回答。

如果回答有價值，說：

```
把這個存到 wiki
```

### 整理（Lint）

定期做健康檢查：

```
幫我整理 wiki
```

LLM 會標記：
- 過時的頁面
- 孤立的頁面（沒有連結）
- 缺失的交叉引用

## 這個系統的價值

- **持久化**：知識建立一次，持續更新
- **交叉引用**：頁面之間自動連結
- **複利增長**：每個新來源讓整個知識庫更豐富
- **你專注學習**：LLM 處理所有書務工作
