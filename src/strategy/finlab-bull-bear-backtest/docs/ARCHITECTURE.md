# 架構設計

## 系統架構

```text
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│ FinLab Data  │─────>│ Regime 判斷   │─────>│ Signal Builder│
│ 收盤價/量/大盤│      │ bull/top/... │      │ Top10 強勢股  │
└──────┬───────┘      └──────┬───────┘      └──────┬───────┘
       │                      │                      │
       v                      v                      v
   [週資料整理]           [30W 均線]             [突破/量增/RS]
                                                      │
                                                      v
                                            ┌────────────────┐
                                            │ BacktestRunner │
                                            │ 訊號延後 + 扣成本│
                                            └──────┬─────────┘
                                                   │
                                                   v
                                            ┌──────────────┐
                                            │ CLI / 報表輸出 │
                                            └──────────────┘
```

## 模組說明

- `data_loader.py`
  - 從 FinLab 取台股收盤價、成交股數與大盤資料。
  - 轉成週資料供策略使用。
- `regime.py`
  - 用 `30` 週均線與短期斜率判斷 `bull`、`top`、`bear`、`base`。
- `signal_builder.py`
  - 只在 `bull` 期間挑股票。
  - 條件是站上 `30` 週均線、量增、突破近 `20` 週高點、相對大盤強勢。
- `cost_model.py`
  - 集中管理手續費與證交稅。
- `backtest_runner.py`
  - 把訊號延後一根 bar 執行，並扣交易成本。

## 資料流範例

1. 讀 FinLab 日資料。
2. 重採樣成週收盤與週量。
3. 判斷大盤是否在牛市。
4. 若是牛市，從全市場選出相對強勢 `Top 10`。
5. 下一根 bar 生效，進入回測。
6. 賣出時扣手續費與證交稅。
